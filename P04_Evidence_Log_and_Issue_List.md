| Evidence ID | Source (แหล่งที่มา) | Description (รายละเอียดข้อเท็จจริงที่พบ) |
|---|---|---|
| EV-01 | คุณเอ (Sales Team) | เปิดดูไฟล์ "ตารางจองห้องประชุมส่วนกลาง" (Excel ใน Shared Drive) เวลา 08:00 น. พบว่าคิว 09:00-11:00 น. ว่าง จึงพิมพ์ชื่อทีมลงไปแล้วกด Save |
| EV-02 | คุณเอ (Sales Team) | ส่งข้อความทาง LINE แจ้งเจ้าหน้าที่ (Admin) ว่าทำการจองห้อง A แล้ว โดยไม่ได้รอให้แอดมินตอบกลับยืนยัน |
| EV-03 | คุณเอ (Sales Team) | พบทีม Project Kickoff นั่งอยู่ในห้องประชุมตอนเวลา 09:00 น. |
| EV-04 | ทีม Project Kickoff | มีหลักฐานเป็น Email ตอบกลับจากแอดมิน เพื่อยืนยันการจองห้อง A ตั้งแต่ช่วงเย็นของวันก่อนหน้า |
| EV-05 | กระบวนการทำงาน | แอดมินรับเรื่องจากหลายช่องทาง (LINE, Email) และต้องมาอัปเดตไฟล์ Excel เองแบบแมนนวล ซึ่งไม่ได้ทำทันทีแบบ Real-time |

## 2. Issue List
รายการปัญหาที่พบจาก Evidence Log นำมาจัดหมวดหมู่เพื่อใช้หาความต้องการของระบบ (Requirements) ต่อไป

| Issue ID | Issue Description (รายละเอียดปัญหา) | Impact (ผลกระทบ) |
|---|---|---|
| ISS-01 | **Decentralized Channels:** ใช้ช่องทางการจองหลายรูปแบบเกินไป (Excel, LINE, Email) | ข้อมูลกระจัดกระจาย แอดมินทำงานซ้ำซ้อน มีโอกาสตกหล่นสูง |
| ISS-02 | **Data Out of Sync:** ข้อมูลในไฟล์ Excel บน Shared Drive ไม่อัปเดตตรงตามเวลาจริง | ผู้ใช้อื่นเห็นว่าห้องว่างทั้งที่โดนจองแล้ว นำไปสู่การจองซ้อนทับกัน |
| ISS-03 | **Unclear Definition of "Booked":** ผู้ใช้ไม่ทราบนิยามและขั้นตอนที่ชัดเจนของการจองสำเร็จ | เกิดความขัดแย้งหน้าห้องประชุม เพราะผู้ใช้คิดไปเองว่าได้ห้องแล้วโดยยังไม่ผ่านการอนุมัติ |

## 3. Fact vs Interpretation
การแยกแยะระหว่าง "ข้อเท็จจริง (Fact)" ที่เกิดขึ้น กับ "การตีความ/ความเข้าใจ (Interpretation)" ของผู้เกี่ยวข้อง เพื่อลดอคติในการออกแบบระบบ

| เหตุการณ์ / สิ่งที่พบ | Fact (ข้อเท็จจริง) | Interpretation (การตีความของ Stakeholder) |
|---|---|---|
| **การจองของทีม Sales** | คุณเอพิมพ์ชื่อลงในไฟล์ Excel และส่งข้อความ LINE หาแอดมินแล้วกดปิด | **ทีม Sales ตีความว่า:** การจองเสร็จสมบูรณ์แล้ว ไม่ต้องรอแอดมินยืนยัน สามารถส่งหมายเชิญลูกค้าได้เลย |
| **สถานะไฟล์ Excel ตอนเช้า** | ไฟล์ Excel ใน Shared Drive ในช่วงเวลา 09:00-11:00 น. เป็นช่องว่างเปล่า | **ทีม Sales ตีความว่า:** ห้องนี้ยังไม่มีใครจอง และไม่มีใครได้สิทธิ์ไปก่อนหน้าตนเองอย่างแน่นอน |
| **การจองของทีม Project** | ทีมส่ง Email จอง และได้รับ Email ตอบกลับยืนยันจากแอดมินแล้ว | **ทีม Project ตีความว่า:** ได้สิทธิ์การใช้ห้องอย่างถูกต้องตามกฎ เพราะมีหลักฐานการอนุมัติจากแอดมินแล้ว |
| **การอัปเดตข้อมูลของแอดมิน** | แอดมินยืนยัน Email ทีม Project ไปแล้ว แต่ยังไม่ได้เปิดไฟล์ Excel เพื่ออัปเดตข้อมูล | **แอดมินตีความว่า:** (คาดการณ์) ตอบ Email ยืนยันแล้วถือว่าจบงานก่อน ส่วนไฟล์ Excel ค่อยมาอัปเดตทีหลังเมื่อมีเวลาว่าง |
